# 一分钱学英语 - 多模型协作提示词设计与任务分配计划

## 1. 项目概述

**项目名称：** 一分钱学英语 / Learn English for one cent  
**目标：** 通过多模型协作实现高效率、低成本的英语学习系统

---

## 2. 模型角色与任务划分

### 2.1 强能力模型（30-40% 工作量）

**适用模型：**
- DeepSeek R1 (Reasoner)
- GPT-4o / GPT-4
- Claude 3.5 Sonnet / Claude 3 Opus
- Gemini 2.5 Pro

**核心任务：**
- 🎯 **战略方向制定：** 整体评估框架设计、评分标准解释
- 🏗️ **框架结构构建：** 评估维度划分、专家系统设计
- 🧠 **复杂逻辑决策：** 跨领域知识整合、深度分析
- ✅ **质量审核与优化：** 输出结果质量校验、逻辑一致性检查

### 2.2 弱能力模型（60-70% 工作量）

**适用模型：**
- DeepSeek Chat (V3.2)
- GPT-4o Mini
- Claude 3 Haiku
- Gemini 2.0 Flash / 1.5 Flash

**核心任务：**
- 📝 **细节内容填充：** 具体语法错误标注、词汇建议
- 🔄 **格式规范化：** 输出格式统一、Markdown 结构整理
- 🔄 **简单重复性工作：** 多轮对话续写、常规问题回答
- 🔄 **翻译与校对：** 中英文互译、拼写检查

```json
{
  "task_id": "writing_eval_001",
  "framework": {
    "sections": ["分项评分", "致命错误", "扣分项", "Band 8.0示范", "核心诊断"],
    "quality_criteria": {
      "accuracy": "high",
      "detail_level": "specific",
      "tone": "constructive"
    }
  },
  "input_data": "...",
  "constraints": {
    "max_tokens": 4000,
    "format": "markdown"
  }
}
```

```json
{
  "task_id": "writing_eval_001",

  "metadata": {
    "model_used": "deepseek-chat",

    "processing_time": "2.3s"
  },
  "flags": ["needs_review", "partial_complete"]
}
```

## 5. 提示词设计要点

### 5.1 强模型提示词模板

**模板结构：**
```
{角色定义}

{战略目标}
- 目标1
- 目标2

{约束条件}
- 约束1
- 约束2

{质量标准}
- 标准1
- 标准2

{任务描述}

请输出：
1. 整体框架
2. 任务分解方案
3. 质量检查清单
```

**写作评估强模型提示词示例：**

```
你是一位资深的雅思写作评估专家，拥有15年以上的教学和评分经验。

战略目标：
- 制定完整的雅思写作评估框架
- 确保评分标准与官方IELTS完全一致
- 识别核心问题，而非表面错误

约束条件：
- 必须基于官方IELTS四项评分标准（TR/CC/LR/GRA）
- 评分必须客观，避免主观偏见
- 诊断必须精准，指出根本原因

质量标准：
- 框架完整性：覆盖所有必要维度
- 诊断准确性：核心问题识别率 > 90%
- 可操作性：提供的建议必须具体可行

任务：
分析用户提交的雅思作文，制定评估框架和任务分解方案。

请输出：
1. 整体框架（5个主要部分）
2. 每个部分的子任务分解
3. 质量检查清单（10项检查点）
```

### 5.2 弱模型提示词模板

**模板结构：**
```
{任务定义}

{执行步骤}
步骤1: ...
步骤2: ...
步骤3: ...

{格式要求}
- 格式1
- 格式2

{参考示例}
示例输入：...
示例输出：...

{原始数据}
{input}

请按照上述要求执行任务。
```

**语法错误标注弱模型提示词示例：**

```
任务：标注作文中的语法错误

执行步骤：
1. 逐句阅读作文
2. 识别语法、用词、搭配错误
3. 对每个错误按指定格式标注
4. 提供修正建议

格式要求：
❌ [错误类型]：原文引用 "[exact quote]"
→ 问题说明：[中文解释]
→ REWRITE: "[corrected version]"

参考示例：
❌ [时态错误]：原文引用 "I go to park yesterday"
→ 问题说明：描述过去的事情应该用一般过去时
→ REWRITE: "I went to the park yesterday"

原始数据：
{user_essay}

请按照上述要求标注所有语法错误。
```

## 7. 实施与评估计划

### 7.1 试点阶段（Week 1-2）

**目标：** 验证3个典型场景的可行性

**任务：**
1. 实现写作评估的多模型协作
2. 实现聊天室专家对话的多模型协作
3. 实现短输入扩写的多模型协作

**评估指标：**
- 输出质量：与单模型对比
- 响应时间：端到端延迟
- 成本降低：API调用费用对比

### 7.2 数据收集与分析（Week 3-4）

**收集数据：**
- 各模型使用频率
- 各模型平均耗时
- 输出质量评分（1-5分）
- 用户反馈收集

**分析维度：**
- 成本效益比
- 质量-成本平衡点
- 瓶颈识别
- 优化机会

### 7.3 优化迭代（Week 5-6）

**优化方向：**
1. 调整任务分配比例
2. 优化提示词设计
3. 简化协作流程
4. 引入缓存机制

**目标：**
- 效率提升：30%+
- 成本降低：20%+
- 质量保持：不低于单模型

### 7.4 全面推广（Week 7+）

**推广内容：**
1. 将多模型协作应用到所有功能模块
2. 建立自动化监控和优化系统
3. 文档化最佳实践
